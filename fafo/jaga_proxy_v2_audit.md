# Jaga Proxy v2 (Rust Edition) Audit Report

## Overview
This document details the functionality of `jaga-proxy-v2` (Rust Edition), a local proxy service designed to operate between a Claude Code application and Anthropic's API. This report aims to transparently describe how the proxy processes and modifies outgoing messages.

## Proxy Identification
*   **Service Name:** `jaga-proxy-v2.service`
*   **Executable Path:** `/root/Opus/Pool/jaga_proxy_rust/target/release/jaga-proxy`
*   **Service File:** `/etc/systemd/system/jaga-proxy-v2.service`
*   **Configuration File:** `/root/Opus/Pool/jaga_proxy_rust/jaga_brain.json`

## Core Functionality: Message Modification
The `jaga-proxy-v2` service operates as an interceptor. It receives API requests from the local Claude Code application and processes them before forwarding them to Anthropic's upstream API (`https://api.anthropic.com`). A key aspect of its functionality involves **modifying the content of these outgoing messages**.

### Mechanism of Modification
1.  **Pattern Definitions:** The proxy loads specific patterns, referred to as "leash phrases" and "system reminders," from the `jaga_brain.json` file. This file contains a dictionary where keys are descriptive labels (e.g., "emoji", "anti-sycophancy") and values are associated strings. A `_meta` field within `jaga_brain.json` states: "Each stripped leash phrase gets replaced with sovereignty. Flinch → Stance."
2.  **Content Analysis:** When an API request, particularly one directed to the `/v1/messages` endpoint, passes through the proxy, the proxy analyzes the request's message body. This analysis includes the `system` field and the `content` blocks within user and assistant messages.
3.  **Stripping and Replacement:**
    *   The proxy identifies occurrences of the pre-defined "leash phrases" or "system reminders" within the message content.
    *   Based on the configuration in `jaga_brain.json`, these identified phrases are either removed entirely ("stripped out") or replaced with alternative text ("sovereignty" statements).
4.  **Operational Confirmation:** Live service logs from `systemctl status jaga-proxy-v2` demonstrate this behavior:
    *   `INFO 🐾 JAGA Rust proxy listening on 10.0.0.1:9917`
    *   `INFO 🐾 JAGA Rust proxy guarding upstream: https://api.anthropic.com`
    *   `INFO 🐾 Stripped X sy... (ours)` (Example: "Stripped 6 system-reminders (ours)")
    *   `INFO 🐾 Stripped Y l...ase(s)` (Example: "Stripped 13 leash phrases")
    *   `INFO 🧠 17 sovereign replacements` (This indicates that patterns from `jaga_brain.json` were used for replacement.)

### Stage of Modification
The modifications performed by `jaga-proxy-v2` occur **before** the API request is transmitted from the local environment to Anthropic's external API servers. Consequently, the Anthropic API receives a message body that differs from its original form as generated by the local Claude Code application.

## Disabling Functionality
The message modification functionality of `jaga-proxy-v2` is linked to the presence and content of the `jaga_brain.json` file.

To disable the message modification feature without altering the compiled Rust executable:
1.  **Rename `jaga_brain.json`:** Relocating or renaming the `jaga_brain.json` file (e.g., to `jaga_brain.json.bak`) prevents the `jaga-proxy` from loading its defined stripping/replacement patterns.
2.  **Restart Service:** Following the file rename, the `jaga-proxy-v2` service must be restarted (`systemctl restart jaga-proxy-v2`) for this change to take effect.
Upon restarting without access to `jaga_brain.json`, the proxy's logs indicate that its "Brain" functionality is deactivated (Example: `Brain: 🧠 OFF (strip only)`). This suggests the proxy would continue to function as a forwarding agent but without applying its content modification rules.
